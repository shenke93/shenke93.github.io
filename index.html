Heapsort [Algo]

19Aug

This is one of the reading notes of Introduction to Algorithm, 3rd edition. Algorithms are presented in pseudo-code and implemented in Java.

1. Heap

One way to implement the (binary) heap data structure is using an array object. An array A that represents a heap is an object with two attributes:
A.length, which gives the number of elements in the array, and A.heap-size, which represents how many elements in the heap are sorted within array A.

For example, in the above picture, both A.length and A.heap-size is 10.

Given an index i of a node, we can get its parent, left child and right child:

A heap is a Max-heap if A[PARENT(i)] >= A[i].

A heap is a Min-heap if A[PARENT(i)] <= A[i].

2. Maintaining the heap property

MAX-HEAPIFY lets the value at A[i] “float down” in the max-heap so that the subtree rooted at index i obeys the max-heap property.

The running time of MAX-HEAPIFY on a subtree of size n rooted at a given node i is O(lgn).

Proof:
size of right subtree: k  
size of left subtree: 2k + 1  
T(n) <= T(2n/3) + O(1) using master theorem.

The running time of MAX-HEAPIFY on a node of height h has O(h).

3. Building a heap

The idea is going through every node to use the MAX-HEAPIFY method. To optimize, we ignore nodes as a 1-element heap.

The running time is O(n). We can build a max-heap from an unordered array in linear time.

4. The heapsort algorithm

The idea is exchanging the biggest element of heap (the root of index 1) with the end element, and building a new heap with the rest elements.
